## 项目概述

本目录为**国内银行流水数据整合**项目，将多家银行的交易流水原始导出文件（CSV/XLS/XLSX）汇总到统一的 Excel 文件中。

## 数据文件说明

| 文件名 | 格式 | 银行 |
|---|---|---|
| 中国银行.csv | CSV（中英双语列头） | 中国银行 |
| 农业银行.xls | XLS | 农业银行 |
| 招商银行.xlsx | XLSX | 招商银行 |
| 浦发银行.xls | XLS | 浦发银行 |
| 中信银行.xlsx | XLSX | 中信银行 |
| 工商银行.xlsx | XLSX | 工商银行 |
| 建设银行.xls | XLS | 建设银行 |
| **国内银行汇总.xlsx** | XLSX | 汇总结果文件 |

## 数据结构

各银行原始文件格式差异较大，以中国银行 CSV 为例：

- **前7行**：账号信息、统计摘要（总笔数、借贷总额、查询时间范围）
- **第8行**：列头（含中英双语字段名）
- **第9行起**：交易明细，主要字段包括：
  - 交易类型（来账/往账）、业务类型、交易日期、交易时间
  - 付款人/收款人账号、名称、开户行
  - 交易金额、交易后余额、交易货币
  - 交易流水号、摘要、用途、附言

## 数据整合目标

将各行异构格式统一映射到 `国内银行汇总.xlsx` 的标准字段，主要整合字段为：
- 银行来源标识
- 交易日期 / 时间
- 借贷方向（借/贷）
- 交易金额
- 余额
- 对手方信息（账号 + 名称 + 开户行）
- 摘要 / 用途 / 附言

## 注意事项

- 各银行原始文件**禁止修改**，仅作为数据源读取
- 汇总文件 `国内银行汇总.xlsx` 为最终输出，处理时以完整数据为准，不得截断
- 中国银行文件为 CSV 格式，其他为 XLS/XLSX，解析时需分别处理
- 金额字段可能含逗号分隔符（如 `6,475.08`）和正负号前缀（`+0.81` / `-400.00`），解析时需清洗

## 生成的脚本文件

| 文件名 | 说明 |
|---|---|
| **整合.py** | 主整合脚本，将各银行源文件原样写入汇总文件子表并更新银行余额 |
| **requirements.txt** | Python 依赖（pandas、openpyxl、xlrd>=2.0.1、chardet） |

### 整合脚本使用方法

1. 将银行源文件重命名为 `{公司代号}-{银行全称}.{扩展名}`，例如 `B-建设银行.xls`
2. 放入本目录（与脚本同目录）
3. 运行：`venv/Scripts/python.exe 整合.py`

### 汇总文件工作表结构

- **Sheet9**：公司代号映射（A-V = 22个公司）
- **银行余额**：按月末日期记录各公司在各银行的期末余额（E列=公司A，Z列=公司V）
- **X-银行缩写**：各公司-银行明细子表（如 `B-建行`、`A-招行`）
