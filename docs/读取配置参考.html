Binance 为「聚合对账」范式(与上面 5 个 1:1 平台不同构):平台无订单号、一个收款 ID 对多笔,故不建查找表逐行追加,而是两侧各按「收款ID + 日期」groupby 求和+计数后 full-outer 比对,产出独立 sheet Binance-USDT对账。admin 侧仅取 其他 以 BIN- 开头且 状态=已完成 的行,USDT 面额取自 奖品名称(正则 USDT\s*([\d.]+),金额列是 TRY 不用);日期口径 t1_window(T+1 窗口对齐)。全部取值/口径由 platforms/5/binance.json 的 recon 声明驱动,详见 平台插件参考。

代号3 平台识别与去重规则(config3.py)

  • 源文件目录:data/input/3/,所有文件平铺放置(均为 .xlsx)
  • 文件识别(stem 小写前缀匹配):admin → admin 订单主表(工作表"汇总");adyen- → Adyen(工作表"Data");华为 → 华为(工作表"Sheet0");googol-/google- → Google Play(第一个工作表,名称含日期后缀)
  • 流水号关联:admin.流水号 ↔ Adyen.Psp Reference / 华为.华为订单号 / Google.Description
  • Adyen 去重:同一 Psp Reference 有多行(Received/Authorised/SentForSettle),按 ADYEN_RECORD_TYPE_PRIORITY = ["SentForSettle", "Authorised"] 优先取 SentForSettle 行
  • Google 去重:只保留 Transaction Type == "Charge" 行(排除 Google fee 和退款行)
  • 平台支付方式填充:Adyen → Payment Method(mc/visa/troy);华为 → 支付方式(World Pay/Adyen/话费);Google → 固定值 "Google Play"
