## 优化记录

### [2026-05-09] 代号3：新增华为月度结算文件支持，完善交易金额汇总手续费列

**内容**
在「交易金额汇总」第一个表格新增「手续费」列：
- Adyen / Google：从每笔订单的 FEE_COL 聚合（原数据已有）
- 华为：从月度结算文件（`华为平台结算-*.xlsx`）按公式计算后汇总
  > 手续费 = 结算金额（结算货币）−（（总交易额 − 销项税额）× 汇率）
- 苹果：暂无手续费数据，保留空白

华为结算文件的 HKD 行**替换**原有 TRY 订单匹配行（合并为单行），避免同(月份, 华为支付)出现两行；笔数从 TRY 行借用后合入 HKD 行。

**原因**
华为平台交易明细文件（`华为平台-*.xlsx`）只含订单金额（TRY/EUR等），无手续费数据。手续费实际由华为月度结算文件提供，但该文件之前被代码主动排除，导致华为手续费一直为空。

**改动文件**
- `src/bank_integration/config3.py`：新增 `HUAWEI_SETTLE_*` 列名常量（6个）
- `src/bank_integration/app3.py`：
  - `scan_source_files_3`：`"华为平台结算"` 前缀识别到新增的 `huawei_settlement` key
  - 新增 `read_huawei_settlement()`：用 `header=1` 读取英文列名
  - 新增 `build_huawei_fee_summary()`：计算手续费，按(月份, 结算货币)聚合
  - `build_summary_sheet`：新增 `huawei_settle_df` 参数，`手续费` 列入 summary_cols，FEE_COL 缺失时安全降级为 0
  - `build_apple_platform_summary`：同步添加空 `手续费` 列避免 concat 后出现 NaN
  - `write_output` / `main`：传递结算数据

**效果**
- 交易金额汇总表增加「手续费」列 ✓
- 华为各月 HKD 结算行出现，手续费数值正确（如 2026-03：-485.3 HKD）✓
- Adyen 手续费来自订单匹配数据，正确聚合 ✓
- 对比表（第二个表格）不受影响，列结构不变 ✓
- 回归测试无新增失败（保持原有 failures=4, errors=2，均为预先存在的测试问题）✓
